## Overview

This document describes how to create a persistent Linux VM (QEMU/KVM) that presents a **shadow view of the host filesystem** with configurable per-path read/write permissions:

- The entire host filesystem is visible inside the VM as **read-only**
- Specific directories you designate are **read-write**, with writes going directly to the host
- Everything is driven by a single config file shared between host and VM
- Mounts can be set up and torn down on demand from the command line

The mechanism is:
1. **NFS** exports the host filesystem — once as read-only (catch-all), and once per RW path (explicitly listed)
2. Inside the VM, the RO root is mounted first, then RW NFS mounts are **bind-mounted** over their corresponding paths in the shadow tree, masking the RO view

---

## Architecture

```
Host OS (Ubuntu)
├── NFS server
│   ├── / exported RO (catch-all shadow)
│   ├── /localdev exported RW
│   └── /home/user/projects exported RW
└── /etc/host-shadow-paths  ← single source of truth

VM (QEMU/KVM, Ubuntu)
├── /mnt/host/              ← RO shadow of entire host
│   ├── /mnt/host/etc       ← read-only
│   ├── /mnt/host/localdev  ← read-write (bind mount over RO)
│   └── /mnt/host/home/user/projects  ← read-write (bind mount over RO)
├── /mnt/rw/                ← staging area for RW NFS mounts
├── /etc/host-shadow-paths  ← synced from host
└── /usr/local/bin/shadow-{mount,umount,sync}
```

---

## Requirements

- Ubuntu 22.04 or 24.04 on both host and VM
- QEMU/KVM with the VM on the default NAT network (`10.0.2.0/24`)
- The host's address from inside the VM is `10.0.2.2` (QEMU default)
- Root/sudo on both host and VM

## Part 1: Host Setup

### 1.1 Install Packages

```bash
sudo apt update
sudo apt install -y nfs-kernel-server
```

### 1.2 Create the Path List

This file is the **single source of truth** for which paths are read-write. Both the host exports and the VM mount script are generated from it.

```bash
sudo tee /etc/host-shadow-paths << 'EOF'
/localdev
/home/greg/projects
EOF
```

Add or remove lines here whenever your RW path set changes. One absolute path per line. Blank lines and lines beginning with `#` are ignored.

### 1.3 Generate /etc/exports

Run this script whenever `/etc/host-shadow-paths` changes:

```bash
sudo tee /usr/local/bin/shadow-export-gen << 'EOF'
#!/bin/bash
set -e

PATHS_FILE=/etc/host-shadow-paths
EXPORTS_FILE=/etc/exports
VM_NETWORK="10.0.2.0/24"

# Write the RO catch-all first
cat > $EXPORTS_FILE << EXPORTS
# Auto-generated by shadow-export-gen — do not edit manually
# RO catch-all: entire host tree visible read-only
/  ${VM_NETWORK}(ro,sync,fsid=0,crossmnt,no_subtree_check)
EXPORTS

# Write one RW export per path
while IFS= read -r path || [[ -n "$path" ]]; do
    # Skip blank lines and comments
    [[ -z "$path" || "$path" =~ ^# ]] && continue
    echo "${path}  ${VM_NETWORK}(rw,sync,no_subtree_check,no_root_squash)" >> $EXPORTS_FILE
done < $PATHS_FILE

exportfs -ra
echo "Exports updated:"
exportfs -v
EOF

sudo chmod +x /usr/local/bin/shadow-export-gen
```

Run it now to generate the initial exports:

```bash
sudo shadow-export-gen
```

Verify:

```bash
sudo exportfs -v
```

Expected output (paths will vary):

```
/                 10.0.2.0/24(ro,sync,wdelay,hide,crossmnt,no_subtree_check,fsid=0,...)
/localdev         10.0.2.0/24(rw,sync,wdelay,hide,no_subtree_check,no_root_squash,...)
/home/greg/projects
                  10.0.2.0/24(rw,sync,wdelay,hide,no_subtree_check,no_root_squash,...)
```

### 1.4 Enable and Start NFS

```bash
sudo systemctl enable --now nfs-kernel-server
sudo systemctl status nfs-kernel-server
```

### 1.5 Firewall (if ufw is active)

QEMU's default NAT network is on `10.0.2.0/24`. Allow NFS from it:

```bash
sudo ufw allow from 10.0.2.0/24 to any port nfs
sudo ufw allow from 10.0.2.0/24 to any port 111
sudo ufw reload
```

## Part 2: VM Setup

These steps run **inside the VM**.

### 2.1 Install Packages

```bash
sudo apt update
sudo apt install -y nfs-common
```

### 2.2 Create Mount Directories

```bash
sudo mkdir -p /mnt/host
sudo mkdir -p /mnt/rw
```

### 2.3 Install the Path List

Copy from the host (run from inside the VM):

```bash
sudo scp user@10.0.2.2:/etc/host-shadow-paths /etc/host-shadow-paths
```

Or create it manually — it must match the host's `/etc/host-shadow-paths` exactly:

```bash
sudo tee /etc/host-shadow-paths << 'EOF'
/localdev
/home/greg/projects
EOF
```

### 2.4 Install the Mount Script

```bash
sudo tee /usr/local/bin/shadow-mount << 'EOF'
#!/bin/bash
set -e

HOST=10.0.2.2
SHADOW_ROOT=/mnt/host
RW_STAGING=/mnt/rw
PATHS_FILE=/etc/host-shadow-paths

if mountpoint -q $SHADOW_ROOT; then
    echo "Shadow already mounted at $SHADOW_ROOT"
    exit 0
fi

echo "Mounting RO shadow of host at $SHADOW_ROOT..."
mount -t nfs4 -o ro,soft,timeo=30 $HOST:/ $SHADOW_ROOT

while IFS= read -r path || [[ -n "$path" ]]; do
    [[ -z "$path" || "$path" =~ ^# ]] && continue

    staging=$RW_STAGING$path
    target=$SHADOW_ROOT$path

    echo "  Overlaying RW: $path"
    mkdir -p "$staging" "$target"
    mount -t nfs4 -o rw,soft,timeo=30 $HOST:$path $staging
    mount --bind $staging $target
done < $PATHS_FILE

echo "Shadow mounted. RO root at $SHADOW_ROOT, RW paths overlaid."
EOF

sudo chmod +x /usr/local/bin/shadow-mount
```

### 2.5 Install the Unmount Script

```bash
sudo tee /usr/local/bin/shadow-umount << 'EOF'
#!/bin/bash
set -e

SHADOW_ROOT=/mnt/host
RW_STAGING=/mnt/rw
PATHS_FILE=/etc/host-shadow-paths

if ! mountpoint -q $SHADOW_ROOT; then
    echo "Shadow is not mounted."
    exit 0
fi

echo "Unmounting RW overlays..."

# Collect paths in reverse order so nested paths unmount before parents
mapfile -t paths < <(grep -v '^\s*#' $PATHS_FILE | grep -v '^\s*$' | tac)

for path in "${paths[@]}"; do
    target=$SHADOW_ROOT$path
    staging=$RW_STAGING$path

    if mountpoint -q "$target" 2>/dev/null; then
        echo "  Unmounting bind: $target"
        umount "$target"
    fi

    if mountpoint -q "$staging" 2>/dev/null; then
        echo "  Unmounting RW NFS: $staging"
        umount "$staging"
    fi
done

echo "Unmounting RO root..."
umount $SHADOW_ROOT

echo "Shadow unmounted."
EOF

sudo chmod +x /usr/local/bin/shadow-umount
```

### 2.6 Install the Sync Script

Keeps the VM's path list in sync with the host when you add or remove RW paths:

```bash
sudo tee /usr/local/bin/shadow-sync << 'EOF'
#!/bin/bash
set -e

HOST=10.0.2.2
PATHS_FILE=/etc/host-shadow-paths

echo "Syncing path list from host..."
scp user@$HOST:/etc/host-shadow-paths $PATHS_FILE
echo "Done. Current RW paths:"
grep -v '^\s*#' $PATHS_FILE | grep -v '^\s*$'
EOF

sudo chmod +x /usr/local/bin/shadow-sync
```

Replace `user` with your actual host username.

## Part 3: Usage

### Mount the Shadow

```bash
sudo shadow-mount
```

### Verify

```bash
# List what is mounted
mount | grep '/mnt/host'

# Try reading a RO path
ls /mnt/host/etc

# Try writing to a RO path — should fail
touch /mnt/host/etc/test 2>&1
# touch: cannot touch '/mnt/host/etc/test': Read-only file system

# Write to a RW path — should succeed
touch /mnt/host/localdev/test-file
ls /mnt/host/localdev/test-file
rm /mnt/host/localdev/test-file
```

### Run Claude Code Against a Host Project

```bash
sudo shadow-mount
cd /mnt/host/localdev/my-project
claude
```

### Unmount the Shadow

```bash
sudo shadow-umount
```

## Part 4: Adding a New RW Path

### On the host

```bash
# 1. Add the path to the source of truth
echo '/home/greg/new-project' | sudo tee -a /etc/host-shadow-paths

# 2. Regenerate exports
sudo shadow-export-gen
```

### In the VM

```bash
# 3. Sync the path list from host
sudo shadow-sync

# 4. If shadow is already mounted, add the new path live
sudo mkdir -p /mnt/rw/home/greg/new-project /mnt/host/home/greg/new-project
sudo mount -t nfs4 -o rw 10.0.2.2:/home/greg/new-project /mnt/rw/home/greg/new-project
sudo mount --bind /mnt/rw/home/greg/new-project /mnt/host/home/greg/new-project

# Or simply unmount and remount to pick up all changes
sudo shadow-umount && sudo shadow-mount
```

## Part 5: Auto-Mount on VM Boot (Optional)

If you want the shadow mounted automatically when the VM starts:

```bash
sudo tee /etc/systemd/system/host-shadow.service << 'EOF'
[Unit]
Description=Host filesystem shadow mount
After=network-online.target
Wants=network-online.target

[Service]
Type=oneshot
RemainAfterExit=yes
ExecStart=/usr/local/bin/shadow-mount
ExecStop=/usr/local/bin/shadow-umount

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable host-shadow.service
```

## Part 6: Troubleshooting

### NFS mount hangs or times out

Check the NFS server is running on the host:

```bash
# On host
sudo systemctl status nfs-kernel-server
sudo exportfs -v
```

Check connectivity from the VM:

```bash
# In VM
showmount -e 10.0.2.2
```

### "Permission denied" when mounting RW path

The path is not listed in the host's `/etc/exports` as a separate RW export. The RO catch-all matches first and the server rejects the `rw` mount request.

Fix: add the path to `/etc/host-shadow-paths` on the host, run `sudo shadow-export-gen`, then re-sync in the VM.

```bash
# Host
echo '/missing/path' | sudo tee -a /etc/host-shadow-paths
sudo shadow-export-gen

# VM
sudo shadow-sync
sudo shadow-umount && sudo shadow-mount
```

### "Already mounted" or stale mounts after VM crash

```bash
# Force unmount everything under /mnt/host
sudo umount -R /mnt/host
sudo umount -R /mnt/rw
```

### Writes fail even on a designated RW path

Confirm the bind mount is in place:

```bash
mount | grep '/mnt/host/localdev'
# Should show two entries: the NFS rw mount at /mnt/rw/localdev
# and the bind mount at /mnt/host/localdev
```

If only the RO NFS root appears, the bind mount step failed. Re-run `shadow-mount` or apply the path live as shown in Part 4.

### Check NFS version in use

```bash
# In VM
nfsstat -m | grep vers
```

All mounts should show `vers=4`. If you see `vers=3`, the NFS server may not have NFSv4 enabled:

```bash
# On host
sudo grep -i nfsvers /etc/default/nfs-kernel-server
# Ensure RPCNFSDARGS does not restrict to v3 only
```

## Reference: File Locations

| File | Host/VM | Purpose |
|---|---|---|
| `/etc/host-shadow-paths` | Both (must match) | Canonical list of RW paths |
| `/etc/exports` | Host | Auto-generated NFS exports |
| `/usr/local/bin/shadow-export-gen` | Host | Regenerates `/etc/exports` from path list |
| `/usr/local/bin/shadow-mount` | VM | Mounts the RO shadow + RW overlays |
| `/usr/local/bin/shadow-umount` | VM | Tears down all mounts cleanly |
| `/usr/local/bin/shadow-sync` | VM | Pulls updated path list from host |
| `/mnt/host` | VM | Shadow root — browse host filesystem here |
| `/mnt/rw` | VM | Staging area for RW NFS mounts (internal use) |
