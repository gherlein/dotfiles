Create an implementation plan for: $@

## Plan Format

For each step:
1. **What**: concrete deliverable
2. **Where**: which files/packages
3. **Dependencies**: what must exist first
4. **Verification**: how to prove it works

## Rules

- Do NOT implement anything. Output only the plan.
- Mark steps that can run in parallel.
- Break work into discrete phases with clear boundaries.
- Check for REQUIREMENTS.md and docs/DESIGN.md first.
- Each phase must have a testable checkpoint.
- If requirements are ambiguous, list the ambiguities and ask.
