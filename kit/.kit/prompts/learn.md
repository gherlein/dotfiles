## Process

1. **Survey**: Read README, PROJECT.md, REQUIREMENTS.md, docs/ if they exist.
2. **Structure**: Map the directory layout and identify key packages/modules.
3. **Dependencies**: Read dependency files (go.mod, package.json, etc.).
4. **Patterns**: Identify the architectural patterns in use.
5. **Build**: Find and read the Makefile or build configuration.
6. **Test**: Find test files and understand the testing approach.
7. **Summarize**: Present findings concisely.

## Rules

- Do NOT modify any code. Read-only exploration.
- Ask clarifying questions if the codebase is ambiguous.
- Note anything unusual or non-standard.
