# Add Emoji

Add appropriate emoji to content in $ARGUMENTS.

## Process

### Pass 1: Add Emoji
- Add emoji to headers, list items, and key terms
- Use contextually appropriate emoji (not random decoration)
- Be consistent: same concept gets same emoji throughout

### Pass 2: Deduplicate
- Review the result for repeated emoji
- Replace duplicates with alternative emoji that fit the context
- Ensure variety while maintaining readability

## Guidelines

- Emoji go at the beginning of headers and list items
- Do not add emoji to inline code or code blocks
- Do not overuse -- one emoji per header/item is sufficient
- Use standard, widely-supported emoji (no obscure symbols)
